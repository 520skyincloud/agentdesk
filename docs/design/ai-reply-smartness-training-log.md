# AI 回复速度与人味评测训练记录

版本：2026-06-30  
范围：企微员工号会话工作台 AI 回复链路  
目标：提高回复速度、人味儿、聪明度，降低错误兜底和媒体上下文丢失。

## 1. 评分公式

每一轮包含 20 条模拟对话。每条对话打分使用如下综合函数：

```text
S_i = 100 * sigmoid(
  0.24 * H_i
  + 0.22 * R_i
  + 0.18 * C_i
  + 0.14 * M_i
  + 0.12 * K_i
  + 0.10 * V_i
  - 0.35 * B_i
  - 0.30 * G_i
  - 0.25 * P_i
)

sigmoid(x) = 1 / (1 + e^(-x))

RoundScore = (Σ S_i * q_i) / (Σ q_i)

q_i = 1 + 0.25 * media_i + 0.25 * service_i + 0.20 * risk_i + 0.15 * burst_i
```

变量说明：

- `H_i`：人味儿，短、自然、不像模板。
- `R_i`：响应速度，普通文本目标 `< 900ms` 进入模型，媒体追问允许短等。
- `C_i`：上下文连续性，能合并短时间连续消息。
- `M_i`：媒体关联能力，图片/语音/文件后接短文本时能结合媒体理解。
- `K_i`：知识边界，知识问题查当前门店知识库，不编造。
- `V_i`：动作安全，服务动作不空口承诺，必要时追问或转人工。
- `B_i`：固定兜底惩罚。
- `G_i`：机器人话术惩罚，例如“根据知识库”“为您”“请稍等片刻”。
- `P_i`：协议/业务风险惩罚，例如把定位发成普通文本、把未理解图片当已看懂。

## 2. 本轮已落地规则

- 文本 debounce：`120ms`。
- 媒体 settle：最长 `900ms`，仅当当前文本明显围绕近期媒体追问时等待。
- 图片/语音/文件后马上补短问题，例如“这个多少钱”“能用吗”“这是什么”，默认当作媒体追问。
- “早餐几点 / 发定位 / 我要办入住 / 送水 / 你好”等独立意图不等待媒体理解。
- 知识库未命中不能固定套话，优先短句追问一个关键细节或转人工。
- 需要真实员工动作的请求不说“已安排/马上送/已通知”。
